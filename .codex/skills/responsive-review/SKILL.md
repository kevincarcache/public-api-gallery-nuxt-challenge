# Responsive Review

## Use this skill when
- A milestone is complete
- A page has complex cards, filters, or detail sections
- Preparing the project for final polish

## Goal
Catch layout problems before they accumulate across many pages.

## Review checklist

### Mobile
- no horizontal overflow
- cards stack cleanly
- filters do not become unusable
- buttons remain tappable
- text remains readable

### Tablet
- grids adapt sensibly
- spacing does not feel too cramped or too sparse

### Desktop
- content width stays readable
- grids do not become visually scattered
- cards remain balanced

## Components to inspect carefully
- homepage card grid
- filter bars
- search inputs
- pagination controls
- image cards
- detail headers

## Anti-patterns
- controls squeezed into a single row on small screens
- over-wide containers
- inconsistent card heights causing broken visual rhythm
- giant empty spaces created by desktop-first layouts

## Done when
- the UI is comfortably usable across common breakpoints
- no major overflow or stacking issues remain
